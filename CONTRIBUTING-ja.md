# コントリビューション

[English](CONTRIBUTING.md)

## 課題報告

課題報告を歓迎します。特に:

- 証明連鎖中の数学的誤りや根拠不足の指摘
- 定理文中の前提が不明確または不足している箇所
- Mathlib 更新に伴う API 破損

## Pull Request

Pull request は以下を満たす必要があります。

- **`sorry` なし。** ビルド (`lake build`) がすべての定理で `sorry` なしに完了すること。
- **前提の明示。** すべての定理は自己完結していること。仮定は型シグネチャに現れなければならない。Mathlib が提供するもの以外の暗黙の公理を導入しないこと。
- **スコープの限定。** 1つの定理ファイルまたは1つの主題につき1 PR。

新しいファイルを追加する場合は `Eigenraum.lean` に import を追加し、両言語の README の証明見取り図を更新してください。

## ビルド

```sh
lake exe cache get
lake build
```

## Mathlib の更新

Mathlib 依存を更新する場合は、既存の全定理がそのままビルドできることを確認してください。API 変更で証明が壊れた場合は同一コミットで修正してください。
